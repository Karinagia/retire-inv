description: Phase 5 — accessibility audit (axe-core + LLM judgment)
argument-hint: [slug or screen]
---

Invoke the `a11y-auditor` subagent to audit accessibility for: ${ARGUMENTS:-the changed screens}

It runs deterministic checks first — **axe-core injected via Playwright** + **Lighthouse** — then
layers LLM judgment over the JSON + DOM + screenshots for the criteria engines can't decide
(meaningful labels, reading order, ARIA on custom widgets, zh-TW contrast on the dark theme).
It enforces P7: WCAG AA contrast, keyboard nav, `focus-visible`, `prefers-reduced-motion`, 44px
targets, no color-only encoding of gain/loss.

It **blocklists high-risk widgets** (drag-drop, custom data grids, toasts, tree views) from
autonomous remediation — those **escalate** to the user instead. Returns: violations (deterministic
+ judgment), drafted fixes, and escalations.
